# 大学城美食地图 - 需求分析文档

---

## 一、项目背景与核心价值

### 1.1 背景痛点

| 痛点维度 | 具体问题 | 影响程度 |
|---------|---------|---------|
| 信息分散 | 美食信息散落在美团、大众点评、小红书、微信群等多个平台 | ⭐⭐⭐⭐⭐ |
| 决策困难 | 学生群体面临"吃什么"的日常选择困难，平均决策时间15-20分钟 | ⭐⭐⭐⭐⭐ |
| 缺乏针对性 | 现有平台面向全人群，缺乏针对学生群体的特色功能（如：性价比、出餐速度、适合聚餐等） | ⭐⭐⭐⭐ |
| 实时性不足 | 不知道商家是否排队、是否满客，到了才发现要等1小时 | ⭐⭐⭐⭐ |
| 社交缺失 | 无法看到"校友都在吃什么"，缺乏可信的口碑参考 | ⭐⭐⭐⭐ |
| 场景模糊 | 不知道某家店适合约会、聚餐还是快餐，试错成本高 | ⭐⭐⭐⭐ |

#### 用户场景细分

| 场景类型 | 典型场景描述 | 核心需求 | 决策因素 |
|---------|-------------|---------|---------|
| **日常快餐** | 中午下课，30分钟内要吃完去上课 | 快、近、便宜 | 距离<500m、出餐<10min、价格<20元 |
| **宿舍聚餐** | 周末4-6人小聚，预算人均50-80 | 氛围好、有包厢/大桌、性价比高 | 评分、环境照、团购优惠 |
| **约会用餐** | 和对象/暧昧对象吃饭，需要氛围 | 环境好、安静、有特色 | 环境评分、私密性、颜值 |
| **深夜食堂** | 晚上10点后饿了，找还在营业的 | 营业时间长、能外卖/到店 | 营业时间、外卖支持 |
| **尝鲜探索** | 想尝试新店或没吃过的菜系 | 真实评价、菜品推荐 | 近期评价、晒图、校友推荐 |
| **外卖决策** | 不想出门，在宿舍点外卖 | 配送快、包装好、满减多 | 配送时间、起送价、活动 |

### 2.3 差异化竞争优势

| 维度 | 美团/大众点评 | 小红书 | 大学城美食地图 |
|-----|-------------|--------|---------------|
| **覆盖范围** | 全城 | 全国/全网 | 聚焦大学城3-5公里 |
| **用户群体** | 全人群 | 年轻女性为主 | 精准大学生 |
| **信息颗粒度** | 商家级 | 单品级 | 商家+场景+实时状态 |

### 3.2 核心功能详细说明

#### 3.2.1 地图浏览模块

| 功能点 | 描述 | 优先级 |
|-------|------|--------|
| **热力地图** | 展示各区域实时客流量热度（红-黄-绿） | P0 |
| **商家标记** | 不同图标区分商家类型（快餐/正餐/饮品等） | P0 |
| **实时状态标记** | 显示"排队中"/"满客"/"空闲"状态 | P0 |
| **区域切换** | 快速切换不同大学周边区域 | P0 |
| **路线规划** | 从当前位置到商家的步行/骑行路线 | P1 |
| **AR导航** | 实景导航（适合复杂商圈） | P2 |

#### 3.2.2 发现推荐模块

| 功能点 | 描述 | 优先级 |
|-------|------|--------|
| **智能推荐** | 基于用户偏好、位置、时间的个性化推荐 | P0 |
| **场景专题** | "适合约会的餐厅"/"深夜食堂"/"人均30吃饱"等 | P0 |
| **校友在吃** | 展示校友最近打卡的店铺（匿名聚合） | P0 |
| **新店速递** | 新开业商家推荐 | P1 |
| **榜单排行** | 口味榜、性价比榜、环境榜、速度榜 | P1 |
| **猜你喜欢** | 基于协同过滤的推荐 | P1 |

#### 3.2.3 商家详情模块

| 功能点 | 描述 | 优先级 |
|-------|------|--------|
| **基础信息** | 地址、电话、营业时间、人均消费 | P0 |

## 七、风险与应对

| 风险类型 | 具体风险 | 应对策略 |
|---------|---------|---------|
| **数据冷启动** | 初期商家/评价数据少 | 运营团队手动录入种子数据；邀请KOC用户首批入驻 |
| **商家配合度** | 商家不愿更新实时状态 | 设计激励机制（曝光加权）；简化更新流程（一键操作） |
| **用户习惯** | 用户已有美团等使用习惯 | 差异化功能（校友口碑、实时状态）；校园地推 |
| **技术实现** | 地图性能、定位精度 | 选用成熟地图SDK；做好降级方案 |
| **内容质量** | 虚假评价、恶意竞争 | 建立审核机制；算法识别异常评价 |
